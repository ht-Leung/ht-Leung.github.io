Project 01 / Agent

scibot

面向长序列实验操作的闭环智能体系统。核心是把实验目标、协议文本、 平台约束和状态反馈编排成可执行、可监测、可恢复的机器人任务图。

  • Task graph
  • Data-driven E2E
  • VLA / WAM

Overview

Scibot 面向长序列实验操作，不把机器人看成单个动作执行器，而是把实验协议、 资源约束、技能调用和状态反馈组织成可执行的闭环任务系统。每个子步骤都被表示为 带输入资源、目标状态、执行技能、成功判据和恢复策略的任务节点。

Architecture

  • Agentic 流程编排层：从实验目标、协议或 SOP 生成任务图、状态机或行为树。
  • 技能执行层：统一接入规则技能、感知-规划-控制技能和 data-driven E2E action model。
  • 感知与观测层：整合多相机图像、机器人状态、设备状态和任务上下文。
  • 状态监测层：判断子步骤是否完成、失败、不确定或需要恢复动作。

Action model

  • 动作层采用 data-driven E2E model，将观测、任务上下文和执行动作放在同一策略接口下。
  • VLA 与 WAM 可作为可替换的策略后端，在相同任务节点、状态判据和日志口径下 benchmarking。
  • 动作结束后通过视觉、机器人状态和任务上下文确认真实实验状态，并触发重试、回退或重新规划。

Project 02 / Platform Infra

HIROLRobotPlatform

运控与数据采集基础设施。主要贡献集中在运动学、IK、OTG 和 impedance control， 为策略 rollout、数据采集和长期真机实验提供稳定执行层。

  • Motion control
  • Data collection
  • Control runtime
Repository github.com/linkage-x/HIROLRobotPlatform git@github.com:linkage-x/HIROLRobotPlatform.git

Built

  • 运动学与 robot model 接口整理。
  • IK 求解链路搭建，用于策略、规划和真机执行之间的目标转换。
  • OTG / Ruckig 执行层，用于平滑追踪高频目标流。
  • Impedance control 接入与真机调试。

Why it matters

Goal conditioned RL、grasping、reward model 和数据采集系统都依赖稳定的底层执行。 这个平台承担的是机器人学习项目里的共同底座，而不是单个 demo 的临时控制代码。

Runtime role

  • 将 robot model、IK、trajectory target 和 controller interface 收敛到统一执行入口。
  • 为高频策略输出提供 OTG / Ruckig 重定时，降低动作突变对真机执行的影响。
  • 同一底层能力可复用于数据采集、policy rollout、在线评测与长期实验。

Project 03 / Real-world RL

goal-conditioned-rl

面向多目标真机策略的在线训练系统。运控基于 HIROLRobotPlatform， 系统侧负责组织 rollout、对象存储、训练任务、策略版本和评测回流。

  • Goal conditioned
  • Online training
  • Object storage

Overview

单任务策略很容易被环境、目标位置和初始化方式锁死。Goal conditioned RL 的重点是把 “目标”显式纳入任务定义，让真机强化学习从固定目标推进到多目标条件下的可泛化执行。

Architecture

  • 底层运控：HIROLRobotPlatform。
  • 执行侧：真机 rollout 持续产生 on-policy trajectories、intervention signals 和 episode metadata。
  • 存储侧：对象存储管理多视角观测、动作序列、奖励信号、任务配置和策略 checkpoint。
  • 训练侧：异步消费新数据，更新 goal-conditioned policy，并将策略版本回流到执行侧评测。

System loop

  • 以多目标任务定义为核心，统一 rollout、training job、policy registry 和 evaluation report。
  • 执行、存储、训练和评测模块解耦，支持不同任务、brain 和 reward source 的组合。
  • 目标是让真机交互数据可以持续进入训练闭环，而不是停留在离线数据集和一次性实验脚本。

Project 04 / VLM Decision

vlm-based-grasping-or-suction

A VLM-driven affordance router for choosing between grasping and suction. The system turns object semantics, mask-level geometry and scene context into an executable manipulation primitive.

  • VLM
  • Affordance routing
  • Robot handoff

Perception prompt

  • 目标 crop、mask、场景图像和必要的几何摘要共同构成 VLM prompt。
  • Prompt 不只描述“是什么物体”，还显式表达目标区域、边界形状、可吸附表面和遮挡关系。
  • 多视角或低置信样本通过投票与一致性检查降低偶发误判。

Decision policy

  • 输出 grasp / suction 选择、置信度、简短 reasoning 和可执行性提示。
  • 决策逻辑结合语义属性、局部几何、接触面质量和末端执行器约束。
  • 目标是把 VLM 的开放世界理解压缩成机器人可消费的离散 action primitive。

Robot handoff

VLM 不直接控制机械臂，而是位于 perception 与 motion execution 之间： 它选择操作模式并提供可解释的决策信号，再交给底层抓取、吸取和可达性检查链路执行。 主要工程约束来自吸嘴硬件能力、深度质量、表面法向估计和极端姿态可达性。

Project 05 / Grasping + VLA

reliable-grasp

Goal-conditioned package grasping system based on Pi05 VLA and Yolo11-Seg visual grounding. 项目重点是把目标条件、数据闭环、策略部署和 benchmark 组织成可复现实验框架。

  • Package grasp
  • Pi05 VLA
  • Benchmark

System

  • 使用 Yolo11-Seg 作为轻量级视觉锚点，将颜色目标、mask、box 和 center 信息输入 Pi05 VLA。
  • Agent / Server 异步推理架构支持真机端实时调用、日志记录和策略切换。
  • 推理侧加入 Kalman filter 与 Temporal Ensembling，降低目标框抖动和长动作 chunk 的累计误差。

Data & infra

  • 提供 package grasp codebase 与 code infra，支撑数据采集、清洗、训练、推理和评测闭环。
  • 指导两名 PRP 学生完成数据构建、策略对比、实验验证和答辩材料整理。
  • 整体数据采集工作已累计有效数据数千小时，PRP / package grasp 部分形成可追踪实验记录。

Benchmark

  • 比较 Prompt-only、Info-mask、Box+Center 等目标条件输入策略。
  • 评估 single-grasp success rate 与 long-horizon progress extent。
  • 保留策略、动作空间、任务版本和失败类型的对照记录，用于回归测试。

Project 06 / Reward Service

dense-reward-model

VLM based dense reward / progress model。项目目标是把 reward 计算从具体任务代码中解耦出来， 做成可支持多种 brain、VLM API 和下游 RL 任务的服务框架。

  • Dense reward
  • VLM API
  • Service framework

Overview

真实机器人强化学习通常卡在 reward 设计和可迁移性上。Dense Reward Model 把任务进度判断、VLM 推理和 reward 输出做成服务层，使不同任务可以共享同一套反馈接口。

Design

  • VLM based progress estimation。
  • 解耦 brain、VLM API 与任务环境。
  • 服务化 reward 输出，支持 RL 训练和在线评估。

Interfaces

  • 提供统一 reward service 接口。
  • 支持通过服务接口接入训练任务、在线评估和模型迭代。

Project 07 / Data System

embodiment-free-data-collection

面向无本体场景的数据采集项目。工作重点是把多视角视觉数据、同步一致性、 采集质量和项目交付组织成稳定的数据生产流程。

  • Project management
  • Multi-camera sync
  • Data quality

Project scope

该项目面向没有机器人本体参与的数据采集场景，核心目标是建立可重复、 可验收、可追踪的数据生产流程。职责范围包括项目 PM、 需求拆解、进度推进和验收口径整理；研发部分主要集中在多路相机同步与 跨流一致性处理。

  • 将同步问题从“后处理对齐”提升为采集链路中的一等数据契约，保证多视角样本在语义和时间上可被稳定消费。
  • 围绕跨相机一致性、录制边界、异常片段和质量标记建立检查逻辑，降低下游训练和 QA 的隐性数据噪声。
  • 输出面向数据闭环的采集单元，使数据能够进入回放、筛选、标注和评测流程，而不是停留在原始视频文件层面。
